Como chegou nisso

Instalação cumulativa via bunx skills add sem nunca remover. O ecossistema de skills do Antigravity empurrou pacotes completos — o Google Workspace veio com 15 core skills, 5 workflows, 35 recipes e 10 personas. Tudo junto, sem distinção do que realmente seria usado.

O custo invisível

Cada skill no system prompt ocupa tokens. Com 118 skills, mesmo que a maioria nunca fosse invocada, seus metadados (nome + descrição) já pesavam no contexto antes de qualquer pergunta. Isso significa menos espaço para o código real e o histórico da conversa.

A cadeia: instalação cega → contexto poluído → respostas piores.

Não era questão de espaço em disco. Era token budget. Cada skill não usado roubava espaço de algo que importa.

Problema

Instalação sem curadoria

Pacotes inteiros de skills eram instalados sem avaliar se 100% do conteúdo seria usado. GWS sozinho trouxe 65 skills.

→
Impacto

Contexto poluído

Metadados de 118 skills no system prompt. Menos tokens para o código da sessão. Respostas menos precisas em projetos grandes como Operações.

→
Solução

Curadoria guiada por uso real

Perguntar por stack, eliminar o que nunca foi usado, manter só o que entrega valor. Resultado: 30 skills, 75% menor.

O que nasceu: eletromidia-artifact.

Da limpeza do contexto, surgiu espaço para criar. Um skill que gera artefatos HTML com o design system da Eletromidia.

Propósito

Gerar artefatos HTML standalone que documentam decisões de arquitetura, implementações e conceitos. Usa o design system Eletromidia: Rethink Sans, paleta vibrante, flat design, sem sombras ou gradientes.

O artefato é auto-contido (só Google Fonts CDN como dependência externa). Abre em qualquer navegador, sem build step.

Gatilhos

/em-artifact, "create artifact", "gerar artefato", "document this", "criar documentação visual". Também sugere proativamente após sessões com mudanças significativas de arquitetura.

Sempre gera em português (pt-BR), a menos que o usuário peça outro idioma.

Estrutura do skill

Design system extraído

Font: Rethink Sans (Google Fonts) Paleta: #FF4F00 orange, #4E18FF purple #FECC14 yellow, #3D7700 green #000000 black, #F577ED pink Design: flat, sem sombras border-radius 18px/14px/12px grid 12 colunas, clamp() fluid Componentes: 24 padrões reutilizáveis hero, cards, tabs, flow board tables, accordions, demo, toast

5 fases. Da sessão ao GitHub Pages.

O workflow do skill foi desenhado para ser interativo: o agente pergunta, o usuário aprova, o código gera.

Fase 1

Coletar contexto

git log, diff, AGENTS.md, Obsidian

Identificar o que foi construído, qual o problema e as decisões tomadas.

Fase 2

Coletar mídia

Screenshots, vídeos, diagramas

Perguntar se o usuário tem arquivos para incluir. Copiar para media/ com nomes limpos.

Fase 3

Estruturar narrativa

Outline com seções e componentes

Mapear conteúdo para o framework: Hero → Contexto → Deep Dives → Demo → Callout.

Fase 4

Gerar HTML

frontend-design skill + design system

Produzir HTML standalone com todos os componentes e interações.

Fase 5

Deploy

gh CLI → GitHub Pages

Push para iskisraell/em-artifacts. Atualizar hub page com novo artefato.

Hub page: o índice de artefatos.

O em-artifacts repo tem uma landing page que lista todos os artefatos com search, time buckets e "Copiar prompt".

Busca

Input de texto que filtra por título, descrição e tags. Case insensitive, resposta instantânea.

Ordenação

Botões "Mais recentes" e "Mais antigos". Default: mais recentes primeiro.

Time buckets

Hoje, Ontem, Última semana, Último mês, Anteriores. Cada seção com contador de artefatos.

Copiar prompt

Botão em cada card que copia um prompt pronto para colar em qualquer agente. O prompt inclui:

  • URL do artefato original
  • Design system constraints (Rethink Sans, paleta, componentes)
  • Contexto do artefato (título, descrição, seções existentes)
  • Espaço para o usuário colar sua solicitação
